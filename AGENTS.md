# AGENTS.md

Repository guidance for Codex and other coding agents working in this tree.

## Project Shape

Tinyblok is ESP-IDF firmware for ESP32-C6. It links a Zig static library into a C ESP-IDF component and publishes sensor data to NATS. The patchbay DSL/runtime is shared conceptually with `lexvicacom/monoblok`; this repo vendors the pure Zig op kernel.

Keep `main/` as the ESP-IDF component root:

- `main/c/` holds C code that touches ESP-IDF, Wi-Fi, sockets, TLS, sensors, and vendored TweetNaCl.
- `main/zig/` holds the patchbay runtime, generated rules, TX ring, Zig entrypoint, and vendored kernel.
- `main/CMakeLists.txt` is the bridge: it registers C sources with ESP-IDF, runs codegen, builds the Zig static library, and links it into the component.

## Common Commands

Use the top-level `Makefile`; it sources ESP-IDF's `export.sh` per command.

```sh
make gen                # regenerate main/zig/rules.zig from patchbay.edn
make test               # zig test main/zig/kernel.zig
make build              # idf.py build
make flash              # build and flash to $(PORT)
make monitor            # serial monitor, exit with Ctrl+]
make flash-monitor      # build, flash, monitor
make menuconfig         # Wi-Fi, NATS, TLS, auth settings
make sync-kernel        # copy kernel.zig from ../monoblok
make sync-kernel-remote # fetch kernel.zig from GitHub
```

`PORT` defaults to `/dev/cu.usbmodem101`. `IDF_EXPORT` defaults to `/Users/alex/esp-idf-v6.0.1/export.sh`.

`make sync-kernel-remote` fetches from `lexvicacom/monoblok` on GitHub. Override `KERNEL_REMOTE_REF=branch-or-sha` to pin a specific upstream revision.

## Source Ownership

- `main/c/stub.c`: `app_main`, NVS, Wi-Fi, NATS connect, driver install, `zig_main`.
- `main/c/nats.c`: NATS client, plaintext/TLS, auth, reconnect behavior.
- `main/c/creds.c`, `main/c/ed25519.c`, `main/c/tweetnacl.c`: creds auth support, only built when enabled.
- `main/c/drivers.c` and `main/c/sources.c`: pump timer/event layer and sensor reads.
- `main/zig/main.zig`: drains TX ring and calls `tinyblok_nats_publish`.
- `main/zig/rules.zig`: generated by `tools/gen.py`; do not hand-edit.
- `main/zig/kernel.zig`: vendored from monoblok; do not patch locally.
- `main/zig/tx_ring.zig`: bounded publish queue.
- `main/zig/patchbay.zig`: glue between generated rules and runtime/kernel types.

## Important Rules

- Do not edit `sdkconfig` for committed defaults; use `sdkconfig.defaults` for checked-in config.
- Never commit real secrets. `sdkconfig` and `secrets/*` are gitignored; `secrets/nats.creds.example` is the placeholder.
- New tinyblok compile-time settings belong in `main/Kconfig.projbuild`, not ad hoc C defines.
- `main/zig/rules.zig` is generated. Update `patchbay.edn` or `tools/gen.py`, then run `make gen` or `make build`.
- `main/zig/kernel.zig` should byte-match monoblok. Make kernel changes upstream, then run `make sync-kernel` or `make sync-kernel-remote`.
- C/Zig interop uses `extern fn` both ways. Undefined-reference linker errors usually mean one side lacks the matching exported symbol.
- Keep ESP-IDF-heavy code in C. IDF macros such as `ESP_ERROR_CHECK`, `WIFI_INIT_CONFIG_DEFAULT`, `IPSTR`/`IP2STR`, event groups, and `ESP_EVENT_DEFINE_BASE` do not translate cleanly through `@cImport`.

## Build Notes

`main/CMakeLists.txt` intentionally lists optional sources eagerly, then applies Kconfig conditionals after `idf_component_register` with `target_sources` and `idf_component_optional_requires`. ESP-IDF's component manager parses the registration call greedily.

Zig target flags must match the ESP32-C6 HP core ABI:

```text
-target riscv32-freestanding-none
-mcpu=generic_rv32+m+a+c+zicsr+zifencei
-O ReleaseSmall
-fno-stack-check
```

If linking fails with ABI-looking errors, compare ESP-IDF's GCC `-march` / `-mabi` invocation and keep Zig aligned.

## Verification

For source/layout changes, run at least:

```sh
make gen
make test
make build
```

`make test` only covers host-portable Zig kernel logic. Firmware behavior still needs `make flash-monitor` on hardware.
